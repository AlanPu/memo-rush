# 记忆力训练游戏应用 Spec

## Why
用户希望通过一系列基于科学研究的记忆力训练游戏来提升自己的认知能力。这些游戏从视觉记忆到工作记忆，难度层层递进，能够系统性地训练不同类型的记忆能力。

## What Changes
- 创建游戏选择主界面，支持下拉菜单选择不同游戏
- 实现第一个游戏：网格翻牌（Corsi Block 变体）
  - 3x3 或 4x4 网格
  - 系统随机闪烁 K 个格子
  - 用户需要凭记忆点击闪烁过的格子
  - 支持关卡递进（闪烁数量递增）
  - 支持干扰项模式

## Impact
- Affected specs: 新增记忆力训练游戏功能
- Affected code: 
  - MainActivity.kt（主界面导航）
  - 新增游戏模块包结构
  - 新增游戏选择UI组件

## ADDED Requirements

### Requirement: 游戏选择主界面
系统应提供一个主界面，允许用户通过下拉菜单选择不同的记忆力训练游戏。

#### Scenario: 用户选择游戏
- **WHEN** 用户打开应用
- **THEN** 显示游戏选择界面，包含下拉菜单
- **WHEN** 用户从下拉菜单选择一个游戏
- **THEN** 导航到该游戏界面

### Requirement: 网格翻牌游戏（Corsi Block 变体）
系统应提供网格翻牌游戏，训练用户的视觉空间工作记忆。

#### Scenario: 游戏初始化
- **WHEN** 用户开始游戏
- **THEN** 显示 N×N 网格（默认 3×3）
- **AND** 从闪烁 2 个格子开始

#### Scenario: 显示闪烁序列
- **WHEN** 关卡开始
- **THEN** 系统随机选择 K 个格子依次闪烁（红色高亮）
- **AND** 闪烁持续约 500ms
- **AND** 闪烁序列结束后恢复原状

#### Scenario: 用户点击记忆位置
- **WHEN** 闪烁序列结束
- **THEN** 用户可以点击网格中的格子
- **WHEN** 用户点击正确的格子
- **THEN** 格子显示正确标记（绿色）
- **WHEN** 用户点击错误的格子
- **THEN** 格子显示错误标记（红色）

#### Scenario: 关卡完成判定
- **WHEN** 用户点击完所有记忆的格子
- **THEN** 系统判断是否全部正确
- **IF** 全部正确
  - **THEN** 进入下一关，闪烁数量 +1
- **IF** 有错误
  - **THEN** 显示失败提示，可选择重试

#### Scenario: 进阶模式
- **WHEN** 用户选择进阶模式
- **THEN** 在闪烁结束后显示干扰项（未闪烁的格子也会高亮提示）
- **AND** 用户需要区分真实闪烁过的格子

### Requirement: 游戏设置
系统应提供游戏设置选项。

#### Scenario: 调整网格大小
- **WHEN** 用户选择网格大小
- **THEN** 可以选择 3×3、4×4 或 5×5 网格

#### Scenario: 重置游戏
- **WHEN** 用户点击重置按钮
- **THEN** 游戏回到第 1 关，闪烁数量重置为 2
